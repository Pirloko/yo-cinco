1. CAPA SEO LOCAL (RANCAGUA) — LANDING PARA GOOGLE
--------------------------------------------------------------------------------

Objetivo: páginas HTML estáticas/servidor para posicionar búsquedas locales de
fútbol en Rancagua, sin alterar la SPA principal (app/page.tsx ni app-context).

Rutas nuevas (Server Components, sin "use client"):

  /rancagua/futbolito          → todos los tipos de partido (filtro "all")
  /rancagua/buscar-rival       → type = rival
  /rancagua/faltan-jugadores   → type = players
  /rancagua/revueltas          → type = open

Archivos de página:
  app/rancagua/futbolito/page.tsx
  app/rancagua/buscar-rival/page.tsx
  app/rancagua/faltan-jugadores/page.tsx
  app/rancagua/revueltas/page.tsx

Cada página incluye:
  - generateMetadata() (title, description, canonical, Open Graph)
  - revalidate = 300 (ISR: datos se refrescan cada ~5 minutos, no solo en build)
  - <h1>, párrafo introductorio (copy SEO), lista de partidos en HTML
  - JSON-LD (schema.org SportsEvent en @graph) vía lib/seo/rancagua-json-ld.ts

Componente compartido (server):
  components/seo/rancagua-seo-page.tsx
  - Lista enlaces a /?matchId=<uuid> por partido
  - Fechas formateadas con lib/match-datetime-format (zona America/Santiago)

Datos desde Supabase:
  lib/supabase/seo-rancagua-matches.ts
  - Tabla: match_opportunities
  - Filtros: status IN (pending, confirmed), date_time > ahora
  - Rancagua: location/venue ILIKE %Rancagua% O city_id en geo_cities (nombre/slug
    Rancagua)
  - Importante: la RLS de anon solo permite leer abiertos (open); para listar
    rival/players en servidor se usa SUPABASE_SERVICE_ROLE_KEY (solo servidor).

Origen canónico del sitio (URLs absolutas):
  lib/seo/site-origin.ts → NEXT_PUBLIC_SITE_URL o fallback https://www.sportmatch.cl

Sitemap dinámico:
  app/sitemap.ts
  - Estáticas: /, las cuatro rutas /rancagua/...
  - Dinámicas: /equipo/[teamId] (teams), /centro/[venueId] (sports_venues)
  - revalidate = 3600
  - Con service role: equipos + centros; sin service role: solo centros (anon puede
    leer sports_venues)

Robots:
  public/robots.txt
  - User-agent: * Allow: /
  - Sitemap: https://www.sportmatch.cl/sitemap.xml
  (Ajustar dominio si el canónico cambia.)

Proxy (deep link sin tocar app-context):
  proxy.ts
  - Petición a /?matchId=<uuid> → redirección a /?joinMatch=<uuid>
  - La SPA ya lee joinMatch y abre el detalle del partido; no se modificó
    lib/app-context.tsx ni app/page.tsx.
